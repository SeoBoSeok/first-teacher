# 까비 어드벤처 — 선생님 버전 (first-teacher)

학생A 게임(noonmin042-source/first)의 fork. 클로드코드를 소프트웨어 공학적으로 모범 사용하는 시범 저장소다.
원본 학생 버전과 비교(diff)가 곧 수업 자료이므로, 기반 기술(Vanilla Canvas)은 바꾸지 않는다.

## 프로젝트 구조

- `kkabbi-adventure.html` — 본편 게임 (플랫포머). 점진적으로 `js/` 모듈로 분리 중
- `index.html` — RETRO CHASER (보조 게임)
- `docs/` — **모든 작업의 기준 문서. 기능 작업 전 반드시 읽을 것**
  - `01-PRD.md` 비전·마일스톤(M1~M4)·하지 않을 것
  - `02-features-scenarios.md` 기능 리스트·수용 기준·시나리오
  - `03-tech-decisions.md` 엔진 결정·목표 구조·조작법
  - `04-teacher-notes.md` 작업 노하우·함정

## 실행·확인

```bash
python3 -m http.server 8200   # http://localhost:8200/kkabbi-adventure.html
```
- 변경 후 반드시 브라우저에서 실행 확인. 확인 전 다음 작업 금지
- 디버그 모드: `?debug=1` (구현 후) — 무적·히트박스 표시

## 작업 규칙

1. **PRD 우선**: 기능 요청이 docs/01-PRD.md 범위 밖이면 만들지 말고 먼저 알릴 것. "하지 않을 것" 섹션 준수
2. **한 번에 한 기능**: 마일스톤(M1→M4) 순서. 여러 기능 동시 작업 금지
3. **수치는 config로**: 새 매직 넘버 금지 — 상수에 이름 붙여 config 영역에 모은다
4. **저장 스키마**: localStorage는 `{ v: N, ... }` 버전 필드 필수, try/catch 필수
5. **점진 분리**: 건드리는 코드만 `js/` 모듈로 빼는 보이스카우트 규칙. 전면 리팩토링 금지
6. **커밋**: 동작하는 최소 단위, 한국어 "무엇을 왜" 형식. 동작 안 하는 코드 커밋 금지
7. 게임 손맛(점프 느낌·속도감)을 바꾸는 수치 변경은 변경 전후 값을 커밋 메시지에 남길 것

## 스타일

- ES Modules, `const` 우선, 함수는 한 가지 일만
- 주석은 "왜"만 — 코드로 보이는 "무엇"은 주석 금지
- 한국어 식별자 금지, 한국어 주석 허용
