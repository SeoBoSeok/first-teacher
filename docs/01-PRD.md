# PRD — 까비 어드벤처 (KKABBI ADVENTURE) 선생님 버전

> 작성: 선생님 버전 저장소(SeoBoSeok/first-teacher) · 원본: 학생A의 noonmin042-source/first fork
> 이 문서가 모든 작업의 기준이다. 기능 추가 전에 반드시 이 문서와 대조한다.

## 1. 제품 비전

직접 그린 캐릭터 4명(Machaei, Mahanni, Mareh, Marisa)으로 던전을 깨는 **레트로 픽셀 플랫포머**.
"내가 만든 캐릭터로, 내가 만든 게임을, 친구가 폰에서 바로 해본다"가 완성의 그림이다.

- 보조 게임: RETRO CHASER (탑다운 아레나 슈터) — 워밍업·실험용
- 플랫폼: 웹 브라우저 (데스크톱 우선 → 모바일 확장)
- 플레이 시간: 1세션 5~15분

## 2. 타깃 사용자

| 사용자 | 시나리오 |
|--------|---------|
| 친구·가족 | 링크 받아서 폰/PC로 바로 플레이. 설치 없음 |
| 제작자(학생A) | 기능을 추가하며 클로드코드·게임 개발을 배움 |

## 3. 현재 상태 (As-Is)

✅ 이미 잘 만들어진 것: 캐릭터 선택(4종, 능력치 차등), 로비+포탈 구조, 점프·더블점프·등반·대시,
적·해저드·스테이지, Web Audio 효과음, 일시정지, 게임오버/클리어 상태.

❌ 완성까지 부족한 것: 진행 저장 없음(껐다 켜면 처음부터), 모바일 조작 불가,
밸런스 미세조정 안 됨, 코드가 한 파일 1,500줄(수정 난도 상승 중), 배포 안 됨.

## 4. 마일스톤 (완성 정의)

| 단계 | 목표 | 완료 기준 (DoD) |
|------|------|----------------|
| **M1. 기억하는 게임** | localStorage 저장 | 최고 기록·클리어한 던전·마지막 캐릭터가 새로고침 후에도 유지 |
| **M2. 손맛과 공정함** | 밸런스·피드백 | 캐릭터 4종이 각자 쓸모 있음. 피격·처치 피드백(화면 흔들림 등) 추가 |
| **M3. 폰에서 한다** | 모바일 터치 | 화면 터치 버튼(이동·점프·공격)으로 전 구간 클리어 가능 |
| **M4. 세상에 공개** | 배포 | GitHub Pages URL로 누구나 접속. 친구 1명 이상의 피드백 수집 |

## 5. 비기능 요구사항

- **성능**: 데스크톱 60fps, 중급 폰 30fps 이상
- **호환**: Chrome·Safari 최신 2개 버전
- **구조**: 빌드 도구 없는 정적 파일 유지 (학생 버전과의 비교 학습 가능성 보존).
  단, 1,500줄 단일 파일은 M1부터 모듈 분리 시작 (`js/` 폴더, ES Modules)
- **저장**: localStorage 스키마에 버전 필드 필수 (`{ v: 1, ... }`) — 구조 바뀌어도 안 깨지게

## 6. 하지 않을 것 (Out of Scope)

- 서버·로그인·온라인 랭킹 (localStorage까지만)
- 새 캐릭터·새 던전 대량 추가 (완성도가 먼저, 콘텐츠는 그다음)
- 프레임워크 도입 (React 등 — 이 게임엔 과잉)
