# 기술 결정 — 참고 게임 · 엔진 선택 · 조작법

## 1. 참고 게임 (무엇을 배워올 것인가)

| 게임 | 배워올 것 |
|------|----------|
| **메이플스토리** | 로비(허브)+포탈 구조 — 이미 까비가 쓰는 구조. "다음 갈 곳"이 보이는 동선 |

## 2. 엔진 선택

### 후보 비교
| 후보 | 장점 | 단점 | 판정 |
|------|------|------|------|
| **Vanilla Canvas (현재)** | 의존성 0, 학생 버전과 1:1 비교 가능, 동작 원리가 전부 보임 | 물리·씬 관리 직접 구현 | ✅ **채택** |
| Phaser 3 | 물리·씬·에셋 관리 내장, 자료 풍부 | 학습 대상이 "엔진 사용법"으로 바뀜, 번들 ~1MB | 보류 |
| Kaboom.js | 간결한 API, 교육용으로 인기 | 커뮤니티 작음, 이미 만든 코드 전면 재작성 필요 | 탈락 |

### 결정: Vanilla Canvas 유지 + ES Modules 분리
- **이유 1 (교육)**: 이 프로젝트의 목적은 게임 출시가 아니라 학생의 성장. 엔진을 쓰면 "왜 움직이는가"가 블랙박스가 된다.
- **이유 2 (연속성)**: 학생 버전과 같은 기반이어야 선생님 버전과의 diff가 곧 수업 자료가 된다.
- **이유 3 (충분함)**: 2D 픽셀 플랫포머에 필요한 물리는 중력+AABB 충돌이 전부고, 이미 구현돼 있다.
- 단, **단일 파일은 졸업한다**: 1,500줄 한 파일 → `js/` 폴더의 ES Modules로 점진 분리 (아래 구조).

### 목표 구조 (M1부터 점진 적용)
```
kkabbi/
├── index.html          # 마크업·CSS만
└── js/
    ├── main.js         # 게임 루프, 상태머신
    ├── config.js       # 모든 상수 (속도·중력·밸런스 수치)
    ├── input.js        # 키보드 + 터치 입력 통합
    ├── save.js         # localStorage 저장/로드 (스키마 버전 포함)
    ├── entities.js     # 플레이어·적
    ├── stages.js       # 로비·던전 데이터
    └── audio.js        # 효과음
```
한 번에 다 옮기지 않는다 — 기능을 건드릴 때 그 부분만 모듈로 빼는 "보이스카우트 규칙".

## 3. 조작법 정의

### 데스크톱 (현재 + 정비)
| 동작 | 키 | 비고 |
|------|----|----|
| 이동 | ← → / A D | |
| 점프 / 더블점프 | ↑ / W / Space | 공중에서 1회 추가 |
| 등반 | 줄·벽에서 ↑↓ | |
| 대시 | Shift | 쿨다운 표시 추가 예정 (M2) |
| 공격 | Enter | |
| 일시정지 | P / Esc | |
| 재시작(게임오버) | Space | 한 키로 즉시 |

### 모바일 (M3 설계)
```
┌────────────────────────────┐
│         게임 화면           │
│                            │
│ ◀ ▶              🅹  🅰    │  ← 왼쪽: 이동 / 오른쪽: 점프(J)·공격(A)
└────────────────────────────┘
```
- 버튼은 화면 하단 모서리, 엄지가 닿는 위치 (쿠키런 배치)
- `touchstart/touchend`를 기존 `keys[]` 상태에 매핑 — 입력 코드를 두 벌 만들지 않는다
- 대시·등반은 모바일에서 단순화 (점프 버튼 더블탭 = 대시)
