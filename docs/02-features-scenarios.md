## 2. 플레이어 시나리오 (유저 스토리)

### S1. 처음 온 친구 (해피 패스)
1. 링크 접속 → 타이틀 화면. "조작법이 한눈에" 보인다.
2. 캐릭터 4명 중 하나 선택 (능력치 비교 표시) → 로비 등장.
3. 로비에서 움직여보며 조작 익힘 → 첫 포탈 진입.
4. 첫 던전은 죽지 않고 깰 수 있는 난이도 → 클리어 → "한 판 더"가 하고 싶어짐.

### S2. 죽었을 때 (실패 루프 — 가장 중요한 시나리오)
1. 던전에서 사망 → GAME OVER + 이번 기록과 **최고 기록**이 같이 보인다.
2. 한 키(또는 한 탭)로 즉시 재도전 — 캐릭터 선택부터 다시 시키지 않는다.
3. 클리어했던 던전은 그대로 클리어 상태다.

### S3. 다시 돌아온 플레이어
1. 다음 날 다시 접속 → 마지막 캐릭터가 선택돼 있고, 클리어 기록이 남아 있다.
2. "지난번에 못 깬 던전"으로 바로 간다.

### S4. 폰으로 하는 친구 (M3)
1. 폰에서 링크 열기 → 화면에 맞게 게임이 보이고 터치 버튼이 뜬다.
2. 버튼 세 개(이동/점프/공격)로 전 구간 플레이 가능.

### S5. 실패 케이스
- localStorage가 막힌 브라우저(시크릿 모드 등) → 저장 없이도 게임은 정상 동작, 조용히 무시.
- 이전 버전 저장 데이터 → 스키마 버전이 다르면 안전하게 초기화 (게임이 깨지지 않는 게 우선).
